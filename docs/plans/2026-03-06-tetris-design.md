# 俄罗斯方块游戏设计文档

## 项目概述

在 `/Tetris` 目录下开发一个现代简约风格的俄罗斯方块游戏，使用 Canvas + 面向对象架构实现。

## 技术方案

### 核心技术栈
- HTML5 Canvas 渲染
- 原生 JavaScript (ES6+)
- CSS3 动画和样式
- 面向对象设计模式

### 架构设计

采用模块化的面向对象架构，主要包含以下核心类：

1. **Game** - 游戏主控制器
   - 管理游戏状态（运行中、暂停、结束）
   - 控制游戏循环（requestAnimationFrame）
   - 协调各个模块的交互

2. **Board** - 游戏棋盘
   - 10列 × 20行的二维数组
   - 管理已落下的方块
   - 检测满行并执行消除

3. **Tetromino** - 方块类
   - 定义7种标准方块（I, O, T, S, Z, J, L）
   - 每种方块有4个旋转状态
   - 处理旋转和移动逻辑

4. **Renderer** - 渲染器
   - 绘制游戏棋盘
   - 绘制当前方块、预览方块、暂存方块
   - 实现消除动画和视觉效果

5. **ScoreManager** - 分数管理器
   - 计算得分和等级
   - 管理最佳成绩（localStorage）
   - 控制难度等级

6. **InputHandler** - 输入处理器
   - 监听键盘事件
   - 处理方块移动、旋转、加速下落

## 游戏功能

### 核心功能
1. **方块移动和旋转**
   - 左右移动：左右方向键
   - 旋转：上方向键或空格键
   - 加速下落：下方向键
   - 瞬间落地：空格键（可选）

2. **下一个方块预览**
   - 显示即将出现的方块
   - 帮助玩家提前规划策略

3. **方块暂存功能**
   - 按 C 键或 Shift 键暂存当前方块
   - 每次下落只能暂存一次
   - 暂存后切换当前方块和暂存方块

4. **最佳成绩记录**
   - 记录不同难度的最高分数
   - 记录最多消除行数
   - 使用 localStorage 持久化存储

5. **难度选择**
   - 简单：初始速度慢（1000ms/格）
   - 普通：初始速度中等（800ms/格）
   - 困难：初始速度快（600ms/格）

### 游戏规则

**消除行计分：**
- 单行消除：100分
- 双行消除：300分
- 三行消除：500分
- 四行消除（Tetris）：800分

**等级提升：**
- 每消除10行提升1级
- 每提升1级，下落速度加快100ms
- 最快速度：200ms/格

**游戏结束：**
- 新方块生成时位置被占用
- 显示最终得分和消除行数

## 视觉设计

### 配色方案
- **背景色：** #1a1a2e（深色调）
- **方块颜色（渐变）：**
  - I方块：青色 (#00d4ff → #0099cc)
  - O方块：黄色 (#ffd700 → #ffaa00)
  - T方块：紫色 (#a855f7 → #7c3aed)
  - S方块：绿色 (#22c55e → #16a34a)
  - Z方块：红色 (#ef4444 → #dc2626)
  - J方块：蓝色 (#3b82f6 → #2563eb)
  - L方块：橙色 (#f97316 → #ea580c)

### Canvas 尺寸
- 主游戏区：300×600px（每格30×30px）
- 预览区：120×120px
- 暂存区：120×120px

### 动画效果
1. **方块下落：** 平滑移动，带轻微阴影
2. **消除行：** 闪烁效果 + 淡出动画（300ms）
3. **方块着陆：** 轻微弹跳效果
4. **游戏结束：** 棋盘从下往上填充灰色

### 布局结构
采用左右布局，与现有的迷宫和扫雷游戏保持一致：
- **左侧：** 游戏棋盘 + 控制按钮
- **右侧：** 预览区 + 暂存区 + 设置面板 + 最佳成绩

## 核心算法

### 碰撞检测
```
检测方块是否可以移动到目标位置：
1. 检查是否超出边界（左、右、下）
2. 检查目标位置是否已有方块
3. 返回是否可以移动
```

### 旋转算法
```
使用旋转矩阵实现：
1. 计算旋转后的新形状
2. 检测碰撞
3. 如果碰撞，尝试墙踢（wall kick）
4. 墙踢失败则取消旋转
```

### 消除行算法
```
1. 从下往上扫描每一行
2. 检测是否满行（所有格子非0）
3. 记录满行索引
4. 播放消除动画
5. 删除满行，上方行下移
6. 更新分数和等级
```

## 开发优先级

1. 基础框架搭建（HTML + CSS布局）
2. Canvas 渲染系统
3. 方块类和旋转逻辑
4. 碰撞检测和移动控制
5. 消除行和计分系统
6. 预览和暂存功能
7. 难度选择和最佳成绩
8. 动画效果和视觉优化

## 技术要点

1. **性能优化：** 使用 requestAnimationFrame 实现流畅渲染
2. **代码复用：** 参考迷宫游戏的 Canvas 渲染经验
3. **用户体验：** 键盘响应灵敏，动画流畅自然
4. **数据持久化：** localStorage 保存最佳成绩
5. **响应式设计：** 适配不同屏幕尺寸

## 测试要点

1. 方块旋转在边界和障碍物附近的表现
2. 消除行的计分准确性
3. 等级提升后速度变化
4. 暂存功能的限制（每次下落只能用一次）
5. 游戏结束条件的准确判断
6. 最佳成绩的正确保存和读取
